# Plan 23-01 — EE-driven BB Package Design

## Contract: `src/ee/bb-design.ts`

```ts
export interface BBDesign {
  /** Matched template (one of mr-base-sln / mr-mod-sln / mr-micro-sln). */
  template: BBTemplateInfo;
  /** OSS-safe NuGet package ids from recipe.uses, minus commercial blocks. */
  packageIds: string[];
  /** Packages filtered out because experience-principles flags them commercial.
   *  Surfaced in UI so user can opt in via --commercial. */
  commercialBlocked: string[];
  /** Top-3 bb-behavioral hints for system-prompt injection during code-gen. */
  behavioralHints: string[];
  /** Raw template description for the form preview. */
  rationale: string;
  /** Top recipe score (0..1). Used by UI to surface low-confidence warning. */
  confidence: number;
}

export async function designBBPackages(
  intent: string,
  opts?: {
    allowCommercial?: boolean;
    eeBaseUrl?: string;
    eeAuthToken?: string;
    signal?: AbortSignal;
  },
): Promise<BBDesign | null>;  // null on EE-down / no template match
```

### Wire format

Same `POST /api/search` body as bb-retrieval.ts:
```json
{ "query": "<intent>", "collections": ["bb-recipes"], "limit": 5 }
```
Note: field name is **`query`** (NOT `text` as bb-retrieval.ts has it wrong —
bb-retrieval works only because the EE server happens to accept both, but the
canonical field is `query`).

### Algorithm

1. Resolve baseUrl + authToken via `getCachedServerBaseUrl()` + `loadEEAuthToken()`.
   If unconfigured → return `null` (caller falls back to manual menu).

2. `Promise.all` 3 queries with `withEeTimeout(p, ms)` from `src/utils/ee-logger.ts`:
   - `bb-recipes` limit=5 (templates + samples mixed)
   - `experience-principles` limit=20 (commercial flags)
   - `bb-behavioral` limit=3 (top hints)

   Default timeout: `MUONROI_BB_DESIGN_TIMEOUT_MS` clamped [500, 5000], default 1500.

3. From bb-recipes results, find entries whose `text` starts with `"Template "`.
   Pick highest-scoring. If none → log `ee-error` once and return `null`.

4. Parse template text:
   - `/\((mr-[a-z-]+)\)/` → shortName
   - `/\|\s*uses:\s*(.+?)(?:$|\n)/` → comma-split → trim each → packageIds
   - Description = text before ` | uses:`

5. Map shortName → `BBTemplateInfo` via reverse-lookup on `NUGET_TO_SHORTNAME`
   constant in `init-new.ts`. (Re-export it or import directly.)

6. Build commercial set:
   - For each principle, regex `/Commercial package (\S+) requires/`
   - Collect into `Set<string>`

7. Split packageIds:
   - If `opts.allowCommercial` → all into `packageIds`, none into `commercialBlocked`
   - Else → partition by commercial set membership

8. behavioralHints = bb-behavioral top-3 text, deduped by exact match (the
   brain currently has 3 duplicate AddInfrastructure entries — dedup defensively).

9. Return BBDesign.

### Failure modes → return `null`

- EE base URL not configured (no ~/.experience/config.json)
- Timeout fired
- HTTP error (any non-2xx)
- No template recipe in top-5 results

All paths must call `logEeFailure("bb-design", classify, err, {elapsedMs, budgetMs})`.
