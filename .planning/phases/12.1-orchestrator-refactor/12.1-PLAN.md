# Phase 12.1: Orchestrator.ts Refactor

> Split `src/orchestrator/orchestrator.ts` (4605 lines) into 7 focused modules

---

## Motivation

The orchestrator file has grown to 4605 lines across a single class (Agent) + 30 standalone functions. This creates:
- **Poor observability** — 43 private fields + 35 public methods + 30 private methods in one file
- **High cognitive load** — council system (744 lines), batch API (350 lines), prompts (200 lines) all in one class
- **Merge conflict magnet** — every feature touches orchestrator.ts
- **Testing friction** — can't unit-test council or prompt logic without instantiating the whole Agent class
- **No logical boundaries** — standalone utility functions at the bottom of the file mix batch, error, tool, and stream concerns

## Success Criteria

1. orchestrator.ts drops from 4605 to ~2500 lines
2. All extracted files pass `bun run typecheck` with zero errors
3. All existing tests pass (`bun test`)
4. Zero behavioral changes — council, batch, processMessage, sub-agent execution all work identically
5. All external imports (src/index.ts, src/ui/app.tsx, src/headless/output.ts) continue to work
