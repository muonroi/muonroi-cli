<objective>
Implement two-pass deliberate compaction engine and `/compact`, `/expand`, `/clear` slash commands. Pass 1 extracts decisions/facts/constraints to `decisions.md`; pass 2 compresses remaining chat within a token budget while preserving `<!-- preserve -->` blocks. `/expand` restores from history snapshots; `/clear` relocks from artifacts.

Purpose: Deliberate compaction never drops decisions (unlike grok-cli's single-pass LLM summarization). History snapshots enable `/expand` reversal. `/clear` enables fresh context from disk artifacts alone.
Output: `src/flow/compaction/` module with extract, compress, preserve, orchestrator. Three slash command handlers. All unit-tested.
</objective>

    1. Create `src/flow/compaction/preserve.ts`:
       - `export const PRESERVE_OPEN = "<!-- preserve -->";`
       - `export const PRESERVE_CLOSE = "<!-- /preserve -->";`
       - `export interface PreservedBlock { id: string; content: string; }`
       - `export function extractPreservedBlocks(text: string): { cleaned: string; blocks: PreservedBlock[] }` — scans for `<!-- preserve -->...<!-- /preserve -->` pairs, replaces each with `__PRESERVED_N__` placeholder, returns cleaned text + array of blocks. Uses regex `/<!-- preserve -->([\s\S]*?)<!-- \/preserve -->/g`.
       - `export function restorePreservedBlocks(text: string, blocks: PreservedBlock[]): string` — replaces `__PRESERVED_N__` placeholders with original content wrapped back in preserve markers.

    2. Create `src/flow/compaction/extract.ts` (pass 1 — deterministic, no LLM):
       - `export interface ExtractedDecisions { decisions: string[]; facts: string[]; constraints: string[]; }`
       - `export function extractDecisions(messages: ModelMessage[]): ExtractedDecisions` — scans all message text content for:
         - Lines matching `/^(?:Decision|Decided):\s*(.+)$/im` -> decisions
         - Lines matching `/^Fact:\s*(.+)$/im` -> facts
         - Lines matching `/^Constraint:\s*(.+)$/im` -> constraints
         - All content inside `<!-- preserve -->` blocks -> decisions (verbatim)
       - Uses `serializeConversation()` from existing engine to get text, then pattern-matches.
       - Returns structured extraction. No LLM call — purely regex-based per RESEARCH.md anti-pattern guidance.

    3. Create `src/flow/compaction/compress.ts` (pass 2 — uses existing engine):
       - `export interface CompressResult { summary: string; preservedBlocks: PreservedBlock[]; tokensAfter: number; }`
       - `export async function compressChat(messages: ModelMessage[], systemPrompt: string, tokenBudget: number, provider: any, modelId: string): Promise<CompressResult>` — orchestrates:
         a. Serialize messages via `serializeConversation()`.
         b. Extract preserved blocks via `extractPreservedBlocks()`.
         c. Estimate tokens of cleaned text. If under budget, return as-is.
         d. Call `prepareCompaction()` + `generateCompactionSummary()` from existing engine for over-budget content.
         e. Restore preserved blocks into the summary via `restorePreservedBlocks()`.
         f. Return result with tokensAfter estimate.

    4. Create `src/flow/compaction/index.ts` (two-pass orchestrator):
       - `export interface CompactionResult { decisionsExtracted: number; tokensBeforeCompress: number; tokensAfterCompress: number; historyPath: string; }`
       - `export async function deliberateCompact(flowDir: string, messages: ModelMessage[], systemPrompt: string, tokenBudget: number, provider: any, modelId: string): Promise<CompactionResult>` — orchestrates:
         a. Pass 1: `extractDecisions(messages)` -> append to `decisions.md` via `writeArtifact()`.
         b. Snapshot: write full serialized chat to `history/<ISO-timestamp>.md` via `atomicWriteText()`.
         c. Pass 2: `compressChat(messages, systemPrompt, tokenBudget, provider, modelId)` -> compressed output.
         d. Return result with metrics.

         b. Call `deliberateCompact(flowDir, messages, systemPrompt, tokenBudget, provider, modelId)`. Note: messages/systemPrompt/provider/modelId must be passed through SlashContext — extend SlashContext if needed by reading from orchestrator state, OR accept that /compact will return instructions for the orchestrator to perform the actual compaction (the slash command triggers the two-pass pipeline, the orchestrator performs it with access to messages).

The slash handler itself validates preconditions (active run exists, flow dir exists) and returns the signal.

       - `export const handleExpandSlash: SlashHandler = async (args, ctx) => { ... }`
       - Logic:
         a. `ensureFlowDir(ctx.cwd)` -> flowDir.
         b. Read `.muonroi-flow/history/` directory. Sort files by name (ISO timestamp = chronological sort). If empty, return `"Nothing to expand. No compaction history found."`.
         c. Read the latest file content.
         d. Delete the snapshot file (per RESEARCH.md Pitfall 5 — prevent double-expand).
         e. Return `"__EXPAND__\n" + content` — the orchestrator intercepts this signal to restore messages from the snapshot. Or return just the file content with a prefix: `"Restored from ${filename} (${lines} lines). Previous compaction reversed."`.
       - Self-register: `registerSlash('expand', handleExpandSlash);`

    3. Create `src/ui/slash/clear.ts`:
       - `export const handleClearSlash: SlashHandler = async (args, ctx) => { ... }`
       - Logic:
         a. `ensureFlowDir(ctx.cwd)` -> flowDir. `getActiveRunId(flowDir)` -> runId.
         b. If no active run: return `"No active run. Nothing to relock from."`.
         c. Load run state. Read decisions.md. Build a relock summary:
            ```
            Relocked from .muonroi-flow/ artifacts.
            Active run: <id>
            Status: <status from state.md>
            Decisions: <count> entries in decisions.md
            Plan: <present/absent from roadmap.md>
            Gray areas: <N open / M total>
            ```
         d. Return `"__CLEAR__\n" + summary` — the orchestrator intercepts to discard chat messages and inject the summary as system context.
       - Self-register: `registerSlash('clear', handleClearSlash);`

    Note on orchestrator integration: The `/compact`, `/expand`, and `/clear` commands return signal-prefixed strings (`__COMPACT__`, `__EXPAND__`, `__CLEAR__`). The orchestrator's `handleCommand` method detects these signals and performs the actual message manipulation (it has access to `this.messages` which slash handlers do not). This is the cleanest boundary — slash handlers validate preconditions and gather state; the orchestrator mutates messages.
