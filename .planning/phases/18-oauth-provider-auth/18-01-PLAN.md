1. **client_id**: Use Codex CLI's published `client_id` (`app_EMznDTI27GiqE5Cz4yviqixP`).
   Rationale: OSS MIT-licensed, already public. Zero registration friction. Documented in code.

2. **Token store layout**: `~/.muonroi-cli/auth/<provider>.json` (per-provider file).
   Rationale: cleaner for future Anthropic/Google extension than a single merged file.

3. **Refresh strategy**: Pre-emptive at `expiresAt - 60_000ms` + 401-retry as defense-in-depth.
   Mutex wraps both paths to prevent double-refresh race.

4. **Userinfo**: Query `https://auth.openai.com/userinfo` after login for email display in `keys list`.

## Test strategy — mock issuer

All OAuth HTTP calls go through an injectable `fetch`-like parameter (default = `globalThis.fetch`).
Tests supply a `mockFetch` that returns scripted responses for:
- `POST /api/accounts/deviceauth/usercode` → `{ user_code, verification_uri, device_code }`
- `GET /api/accounts/deviceauth/usercode?device_code=...` → first `{ status: "pending" }` then `{ status: "complete", authorization_code }`
- `POST /oauth/token` (exchange) → `{ access_token, refresh_token, id_token, expires_in }`
- `POST /oauth/token` (refresh) → updated tokens
- `POST /oauth/revoke` → 200
- `GET /userinfo` → `{ email, name }`

No live network calls. `auth.openai.com` never hit in CI.
