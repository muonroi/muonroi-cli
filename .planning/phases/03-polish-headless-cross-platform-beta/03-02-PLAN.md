4. Test case 1 — "emits valid JSONL step_start / text / step_finish for simple prompt":
   - Create emitter with sessionId "golden-test-1"
   - Call `observer.onStepStart({ stepNumber: 1, timestamp: 1000 })`
   - Call `consumeChunk({ type: "content", content: "Hello world" })` — collect stdout lines
   - Call `observer.onStepFinish({ stepNumber: 1, timestamp: 2000, finishReason: "stop", usage: { inputTokens: 10, outputTokens: 5 } })`
   - Call `flush()` — collect remaining lines
   - Parse ALL collected lines as JSON
   - Assert: at least one event has `type === "step_start"` and `stepNumber === 1`
   - Assert: at least one event has `type === "text"` and `text === "Hello world"`
   - Assert: at least one event has `type === "step_finish"` and `finishReason === "stop"`
   - Assert: every event has `sessionID === "golden-test-1"`

5. Test case 2 — "emits tool_use events for tool call chunks":
   - Create emitter, trigger onStepStart
   - Feed `consumeChunk({ type: "tool_calls", toolCalls: [{ id: "tc1", type: "function", function: { name: "read_file", arguments: '{"path":"test.ts"}' } }] })`
   - Call `observer.onToolStart?.({ toolCall: { id: "tc1", type: "function", function: { name: "read_file", arguments: '{"path":"test.ts"}' } }, timestamp: 1500 })`
   - Trigger onStepFinish + flush
   - Assert: at least one event has `type === "tool_use"` or the tool call is represented in the output

6. Test case 3 — "handles empty session (no chunks)":
   - Create emitter, onStepStart, immediately onStepFinish + flush
   - Assert: step_start and step_finish events present, no text event
