## Tasks

### T1 — Hash upgrade
- `src/orchestrator/cross-turn-dedup.ts:61` — rewrite `shortHash`:
  ```ts
  function shortHash(text: string): string {
    return createHash("sha256").update(text).digest("hex").slice(0, 16);
  }
  ```
- Update line 117 stub format to `sha256=${hash}`.
- Add a one-line comment above `shortHash` explaining collision property: 16 hex = 64 bits → birthday collision at ~4B entries; LRU cap is 200, so overkill-safe.

### T2 — Fix existing unit test
- `src/orchestrator/__tests__/cross-turn-dedup.test.ts:24` — change `toContain("sha1=")` to `toContain("sha256=")`.

### T3 — Add collision regression test
Append to the same test file:
```ts
it("distinct large tool outputs do not collide on sha256-16", () => {
  // Two large strings differing only in last char must produce different hashes
  const a = "A".repeat(2000);
  const b = "A".repeat(1999) + "B";
  // Use the registry through two distinct turns, assert content stays distinct
  ...
});
```
Goal is regression coverage on the property "different inputs → different stubs", not a Bloom-style probabilistic test.

### T4 — Create `tests/harness/cost-leak-c3.spec.ts`
Model on existing `cost-leak-b3.spec.ts`:
- Install mock-model with two-turn fixture.
- Both turns issue the same `read_file` tool call on the same path → identical tool_result.
- Assert second turn's `tool_result` in the recorded prompt contains `[tool_result identical to earlier turn — dedup ref sha256=`.
- Assert dedup hits do NOT collide across two genuinely different file reads in a separate sub-test.

### T5 — Shim deprecation warning
- `src/agent-harness/index.ts` — add a one-shot warning gated by an env var:
  ```ts
  if (
    typeof process !== "undefined" &&
    process.env.MUONROI_INTERNAL_SHIM_OK !== "1" &&
    !(globalThis as { __muonroiShimWarned?: boolean }).__muonroiShimWarned
  ) {
    (globalThis as { __muonroiShimWarned?: boolean }).__muonroiShimWarned = true;
    console.warn(
      "[muonroi] DEPRECATED: importing from 'src/agent-harness' is internal-only. " +
      "External consumers must import from '@muonroi/agent-harness-core' or " +
      "'@muonroi/agent-harness-opentui'. See CHANGELOG.md migration section."
    );
  }
  ```
- Add `MUONROI_INTERNAL_SHIM_OK=1` to:
  - `vitest.config.ts` and `vitest.harness.config.ts` (env block).
  - The harness `spawnHarness()` helper in `tests/harness/helpers.ts` (passes through to child).
  - Any internal launcher script in `scripts/` that runs from this repo.

## Out of scope

- Removing the shim entirely (would force a large internal refactor).
- Migrating internal callers off `src/agent-harness/*` subpath imports.
- Bumping package versions to 0.1.0 (publish workflow is a separate decision).

## Risks

- **Cache key invalidation:** clients that persisted dedup keys across sessions (none today, but possible) would see cache misses post-deploy. Acceptable — dedup is session-scoped per `src/orchestrator/cross-turn-dedup.ts:66`.
- **Deprecation warning noise:** harness specs spawn the TUI process; if `MUONROI_INTERNAL_SHIM_OK=1` is missing in any one launcher, the noise floods CI logs. Sweep all spawn helpers in T5.
