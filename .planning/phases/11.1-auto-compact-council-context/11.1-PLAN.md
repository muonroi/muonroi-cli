# Phase 11.1: Auto-Compact Detail Preservation & Council Context Injection

## Goal

Fix 2 quality gaps found after Phase 11:
1. **Compact summary mất detail** — `TOOL_RESULT_MAX_CHARS = 2000` quá thấp, summarization prompt chưa tối ưu cho việc giữ lại thông tin quan trọng (file paths, errors, số liệu)
2. **Council thiếu context** — Khi user gõ `/council`, các agent không biết conversation history, dẫn đến phân tích thiếu căn cứ và lan man

## Depends on
- Phase 11 (auto-compact visibility đã ổn định)

---

## Problem Analysis

### Problem 1 — Compact summary quá lossy (Impact: HIGH)

**Root cause 1a: `TOOL_RESULT_MAX_CHARS = 2000`**
- Tool results (file reads, grep output, bash stdout) bị cắt ở 2000 chars
- Với file code dài 500 dòng → mất gần hết nội dung
- Agent ở turn sau không biết chính xác code đang nói gì

**Root cause 1b: Summarization prompt chỉ là mô tả chung**
- Prompt bảo "preserve exact file paths, function names, and error messages"
- Nhưng không có cấu trúc rõ ràng để buộc model làm điều đó
- Không có section riêng cho "Critical Data" (số liệu, đường dẫn, lỗi)

**Root cause 1c: `serializeConversation()` xử lý tool result thô**
- Chỉ gọi `truncateForSummary()` rồi bỏ vào `[Tool result]: ...`
- Không có cấu trúc để phân biệt file paths vs thông tin thường

**Current flow:**
```
Old messages → serializeConversation() → LLM summarize → summary message (mất detail)
```

### Problem 2 — Council thiếu conversation context (Impact: HIGH)

**Root cause 2a: `/council` chỉ truyền user text làm `topic`**
```typescript
// src/ui/slash/council.ts:13
const topic = args.join(" ") || ctx.lastPrompt || "";
```

`ctx.lastPrompt` chỉ là prompt cuối cùng, không phải toàn bộ conversation.

**Root cause 2b: `_buildDiscussPrompt("open", ...)` chỉ thấy `topic`**
```typescript
prompt: `Topic for discussion:\n${ctx.topic}`,
```

Agent không biết:
- User đã làm gì trước đó?
- Đã có quyết định/plan nào chưa?
- Context của dự án là gì?
- Outcome cần đạt là gì?

**Root cause 2c: Leader synthesis không có structured output requirement**
- Leader được yêu cầu synthesize nhưng không có format rõ ràng cho outcome
- Council xong → user vẫn phải tự extract kết luận → thủ công

**Current council flow:**
```
/council <topic>
  → Phase 1: Mỗi agent phân tích chỉ dựa trên <topic>
  → Phase 2: Debate qua lại
  → Phase 3: Leader synthesis (không structured)
  → Kết thúc, không inject kết quả vào conversation
```

## Solutions

### Fix 1: Preserve detail trong compact summary

**1a. Tăng `TOOL_RESULT_MAX_CHARS` từ 2000 → 8000**
- File: `src/orchestrator/compaction.ts`
- 8000 chars (~200 dòng code) đủ cho hầu hết file trung bình
- Có thể configurable qua settings về sau

**1b. Cấu trúc hóa summarization prompt**
- File: `src/orchestrator/compaction.ts`
- Thêm section `## Critical Data` trong prompt output format
- Bắt buộc model liệt kê:
  - File paths đã được đề cập
  - Function/method names
  - Error messages (nguyên văn)
  - Số liệu/queries kết quả
- Giữ nguyên format cũ, chỉ thêm section

**1c. Cải thiện `serializeConversation()` để preserve structure**
- File: `src/orchestrator/compaction.ts`
- Thêm prefix rõ ràng cho từng loại message:
  - `[User]:` (giữ nguyên)
  - `[Assistant response]:`  
  - `[Tool call]: <tool_name>(<args>)`
  - `[Tool result from <tool_name>]: <content>` (kèm tên tool)
- Giúp LLM summarization hiểu context của từng message

### Fix 2: Inject conversation context vào council

**2a. Mở rộng `_buildDiscussPrompt()` để nhận conversation context**
- File: `src/orchestrator/orchestrator.ts`
- Thêm tham số `conversationContext?: string` vào DiscussPromptContext
- Nếu có, prepend vào system prompt:
  ```
  ## Conversation Context (before this discussion)
  <context>
  ```
- Không thay đổi logic `topic` hiện tại, chỉ bổ sung context

**2b. Build conversation context từ messages hiện tại**
- File: `src/orchestrator/orchestrator.ts`
- Method mới: `_buildCouncilContext(): string`
- Trích xuất từ `this.messages`:
  - User's goal (từ user messages gần đây)
  - Compaction summary (nếu có)
  - Key decisions đã được ghi nhận
  - Current state (đang làm gì)
- Giới hạn ~3000 tokens để không tốn cost

**2c. Primary agent (leader) inject outcome requirement**
- File: `src/orchestrator/orchestrator.ts`
- Thêm `outcomeRequest` vào council context:
  ```
  ## Required Outcome
  After discussion, the leader must produce a structured result:
  1. AGREED: [points all participants agree on]
  2. TRADE-OFFS: [genuine disagreements remaining]
  3. RECOMMENDATION: [leader's final call]
  4. NEXT STEPS: [concrete actions with owners]
  ```
- Leader synthesis prompt được cập nhật để output theo format này

**2d. Inject council outcome vào conversation sau khi kết thúc**
- File: `src/orchestrator/orchestrator.ts`
- Sau Phase 3, thay vì chỉ yield content, append outcome như system message:
  ```typescript
  appendSystemMessage(this.session.id, `[Council Outcome] ${structuredOutcome}`);
  ```
- Để các turn sau có context từ council

### Fix 3: Tăng `keepRecentTokens` cho model có context window lớn (Nice-to-have)

- File: `src/orchestrator/orchestrator.ts`
- `getCompactionSettings()`: nếu contextWindow > 200K, tăng `keepRecentTokens` lên 10% của window
- Giúp model có window lớn (1M+) giữ được nhiều recent context hơn

## Implementation Plan

### Wave 1 — Compact detail preservation
Files: `src/orchestrator/compaction.ts`

1. Tăng `TOOL_RESULT_MAX_CHARS` từ 2000 → 8000
2. Thêm `TOOL_RESULT_MAX_CHARS_CONFIGURABLE` — export hằng số để sau này có thể setting
3. Cập nhật `SUMMARIZATION_PROMPT`: thêm section `## Critical Data` vào format
4. Cập nhật `serializeConversation()`: thêm `[Tool call]` prefix riêng, kèm tên tool trong `[Tool result]`

### Wave 2 — Council context injection
Files: `src/orchestrator/orchestrator.ts`, `src/ui/slash/council.ts`

5. Thêm interface `DiscussPromptContext` mở rộng với `conversationContext?: string`
6. Implement `_buildCouncilContext()` method:
   - Lấy compaction summary message nếu có
   - Lấy user messages gần nhất (3-5 messages cuối)
   - Lấy key decisions từ memory nếu có
   - Format thành string tối đa 3000 tokens
7. Update `_buildDiscussPrompt()`:
   - Nhận thêm `conversationContext` trong ctx
   - Nếu có, prepend vào system prompt
8. Update `runCouncilRound()`:
   - Build context từ `_buildCouncilContext()` trước phase 1
   - Pass context vào `_buildDiscussPrompt()` cho tất cả phases

### Wave 3 — Structured outcome & injection
Files: `src/orchestrator/orchestrator.ts`

9. Cập nhật `SUMMARIZATION_PROMPT` cho leader synthesis (Phase 3):
   - Yêu cầu output theo format:
     ```
     ## AGREED
     ...
     ## TRADE-OFFS
     ...
     ## RECOMMENDATION
     ...
     ## NEXT STEPS
     ...
     ```
10. Parse structured outcome từ leader synthesis text
11. Append outcome vào session memory dưới dạng system message:
    ```
    [Council Outcome]
    Topic: <topic>
    Recommendation: <recommendation>
    Next Steps: <steps>
    ```

### Wave 4 — Adaptive keepRecentTokens (nice-to-have)
File: `src/orchestrator/orchestrator.ts`

12. Update `getCompactionSettings()`: nhận thêm `contextWindow?: number`
13. Nếu `contextWindow > 200_000`: `keepRecentTokens = Math.floor(contextWindow * 0.1)`
14. Giới hạn `keepRecentTokens` trong [20_000, 100_000]
