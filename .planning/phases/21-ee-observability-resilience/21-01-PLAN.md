## Tasks

### T1 — Create `src/utils/ee-logger.ts`
Minimal wrapper:
```ts
export interface EeFailureExtra { elapsedMs?: number; budgetMs?: number; [k: string]: unknown }
export function logEeFailure(source: string, kind: "timeout" | "error", err: unknown, extra?: EeFailureExtra): void
```
Behaviour:
1. Always `console.warn(\`[ee.${source}.${kind}]\`, { name, message }, extra)` with structured shape.
2. If `(globalThis as any).__muonroiAgentRuntime?.emitEvent` is a function, emit `{ t: "event", kind: kind === "timeout" ? "ee-timeout" : "ee-error", source, elapsedMs, budgetMs, ts: Date.now() }`.
3. Pure aside from log + event. Never throws.

Add unit tests at `src/utils/__tests__/ee-logger.test.ts`:
- emits warn with expected source/kind formatting
- emits event when runtime present
- no-op event path when runtime absent
- never throws on `null`/`undefined`/string err

### T2 — Extend protocol with `ee-timeout` / `ee-error`
- `packages/agent-harness-core/src/protocol.ts` — add two `LiveEvent` discriminants. Payload:
  ```ts
  | { kind: "ee-timeout"; source: string; elapsedMs?: number; budgetMs?: number; ts: number }
  | { kind: "ee-error"; source: string; name?: string; message?: string; ts: number }
  ```
- Bump protocol minor: `0.2.0` → `0.3.0` (find the version constant via grep; update single source of truth).
- Update CLAUDE.md event kinds table (around lines 257-272) — append two rows; do NOT rewrite the entire table.

### T3 — Refactor 12 silent catches
For each row in the table above, swap `.catch(() => <fallback>)` for:
```ts
.catch((e) => {
  logEeFailure("<source>", classifyEeError(e), e);
  return <existing-fallback>;
})
```
Add a small helper `classifyEeError(e)` in `ee-logger.ts`:
```ts
export function classifyEeError(e: unknown): "timeout" | "error" {
  const name = (e as { name?: string } | null)?.name ?? "";
  return name === "TimeoutError" || name === "AbortError" ? "timeout" : "error";
}
```

## Risks

- Protocol bump may break any consumer asserting on the kind enum exhaustively — grep `LiveEvent` and event kind switches in `tests/harness/` and `packages/` before bumping. Additive change so most code paths will not need updates.
- Adding logs to fire-and-forget paths could increase console noise during normal degraded operation. Acceptable — the whole point is observability. If noise becomes a real complaint, add a `DEBUG_EE` gate in a follow-up phase, not now.
