  <files>src/ee/bridge.ts, src/ee/bridge.test.ts, src/ee/index.ts</files>
  <behavior>
    BRIDGE-01 — Typed facade:
    - classifyViaBrain(prompt, timeoutMs?) returns Promise<string | null>
    - searchCollection(name, vector, topK, signal?) returns Promise<EEPoint[]>
    - routeModel(task, context, runtime) returns Promise<EERouteResult | null>
    - routeFeedback(taskHash, tier, model, outcome, retryCount, duration) returns Promise<boolean>
    - getEmbeddingRaw(text, signal?) returns Promise<number[] | null>
    - resetBridge() resets singleton for test isolation

    BRIDGE-02 — Graceful degradation:
    - When fs.access rejects (file not found): all functions return null/[]/false, console.warn logs one-line message containing "experience-core.js not found"
    - When createRequire throws (corrupt file): all functions return null/[]/false, console.warn logs one-line message containing "failed to load"
    - Load is attempted only once (lazy singleton with _loadAttempted flag)
    - resetBridge() clears the singleton so tests can re-trigger load

    BRIDGE-03 — Config isolation:
    - No function signature accepts qdrantUrl, ollamaUrl, or brainModel params
    - bridge.ts does not import from src/ee/auth.ts or read config.json
    - bridge.ts does not set any EXPERIENCE_* environment variables
  </behavior>
  <implementation>
    RED phase: Write bridge.test.ts with vitest mocking createRequire and fs.access. Tests cover:
    1. All 5 functions callable and return correct types when core loaded
    2. All 5 functions return null/[]/false when core absent (fs.access rejects)
    3. All 5 functions return null/[]/false when require throws
    4. console.warn called with descriptive message on degradation
    5. Lazy singleton: second call does not re-attempt load
    6. resetBridge clears state for re-attempt

    GREEN phase: Implement bridge.ts following the research skeleton exactly:
    - createRequire(import.meta.url) for CJS loading

    - resetBridge() for test isolation
