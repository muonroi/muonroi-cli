async cleanup(): Promise<void> {

async clearHistory(): Promise<void> {

**IMPORTANT — clearHistory callers audit:**

There are 2 callers:

Since `clearHistory()` now returns `Promise<void>`, calling without `await` means the promise floats. However, the critical requirement is that extraction fires BEFORE `startNewSession()` resets `this.messages`, which is guaranteed because `await extractSession(...)` is INSIDE `clearHistory`. The caller does NOT need to await `clearHistory()` for correctness — the extraction is self-contained within the method.

2. `src/ui/app.tsx` line ~2146: `agent.startNewSession()` — This calls startNewSession directly, NOT clearHistory. No change needed.

No TypeScript errors expected — changing `void` return to `Promise<void>` is compatible at call sites that don't use the return value.
