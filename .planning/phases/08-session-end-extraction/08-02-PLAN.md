clearHistory(): void {

With:
```typescript
async clearHistory(): Promise<void> {
  // D-09: Extract messages accumulated since last clear BEFORE reset
  await extractSession(this.messages, this.bash.getCwd(), "cli-clear", this.getSessionId())
    .catch(() => {}); // D-05: redundant safety — extractSession already swallows
  this.startNewSession();
}
```

**IMPORTANT — clearHistory callers audit:**

There are 2 callers:
1. `src/ui/app.tsx` line ~2490: `agent.clearHistory()` — This is in an async function body (`processSlashCommand` or similar). Since `clearHistory()` now returns `Promise<void>`, calling without `await` means the promise floats. However, the critical requirement is that extraction fires BEFORE `startNewSession()` resets `this.messages`, which is guaranteed because `await extractSession(...)` is INSIDE `clearHistory`. The caller does NOT need to await `clearHistory()` for correctness — the extraction is self-contained within the method.

2. `src/ui/app.tsx` line ~2146: `agent.startNewSession()` — This calls startNewSession directly, NOT clearHistory. No change needed.

No TypeScript errors expected — changing `void` return to `Promise<void>` is compatible at call sites that don't use the return value.
