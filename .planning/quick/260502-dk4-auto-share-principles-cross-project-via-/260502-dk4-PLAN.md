2. In `buildScope()`, after `const { remote } = parseGitConfig(cfg);` (line 56) and before the branch/repo/global assignment (lines 58-60), add ecosystem detection:

```typescript
// Ecosystem detection — check if remote matches any configured pattern
if (remote) {
  const eco = loadUserSettings().ecosystem;
  if (eco && eco.patterns.some((p) => remote.includes(p))) {
    cached = { kind: "ecosystem", name: eco.name };
    cachedFor = opts.cwd;
    return cached;
  }
}
```

This runs BEFORE the branch/repo fallback. If a match is found, the ecosystem scope takes priority.
If no ecosystem config or no pattern matches, falls through to existing repo/branch/global logic.

The loadUserSettings() call is cheap (reads from disk once, or already cached by the process).
Scope itself is cached per session via the existing `cached`/`cachedFor` mechanism, so ecosystem detection only runs once per cwd.
